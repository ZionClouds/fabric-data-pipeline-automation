  Before agents can "think", we need to define everything required to build a complete pipeline:

  ┌─────────────────────────────────────────────────────────────────────────┐
  │                    COMPLETE PIPELINE REQUIREMENTS                        │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                          │
  │  1. SOURCE INFORMATION                                                   │

  Agent Thinking Model: "What Do I Need to Know?"

  Each agent should have a knowledge checklist - things it MUST know before it can do its job:

  Discovery Agent's Checklist

  DISCOVERY_AGENT_REQUIREMENTS = {
      "must_know": [
          "What is the source system?",
          "What is the end goal/use case?",
          "Who will consume this data?",
          "Is this one-time or ongoing?",
      ],
      "should_know": [
          "What's the business context?",
          "Any deadline or urgency?",
          "Is this POC or production?",
      ],
      "nice_to_know": [
          "Budget constraints?",
          "Team expertise level?",
      ],

      "thinking_rules": [
          "If user says 'migrate' → ask if they need ongoing sync too",
          "If user mentions 'customer data' → flag potential PII",
          "If user says 'Power BI' → will need aggregated layer",
          "If user says 'real-time' → different architecture entirely",
      ]
  }

  Source Analyst's Checklist

  SOURCE_ANALYST_REQUIREMENTS = {
      "must_know": [
          "Exact source type and version",
          "Cloud or on-premise?",
          "What objects to extract (tables/files/APIs)?",
          "Estimated data volume",
      ],
      "should_know": [
          "Schema details (columns, types)",
          "Primary keys / relationships",
          "How data changes (insert-only, updates, deletes)?",
          "Any CDC/change tracking available?",
      ],
      "nice_to_know": [
          "Current performance issues?",
          "Network constraints?",
          "Existing ETL tools in use?",
      ],

      "thinking_rules": [
          "If on-premise → MUST have gateway, check if exists",
          "If PostgreSQL → ask about logical replication for CDC",
          "If volume > 100GB → need partitioning strategy",
          "If tables have FKs → need load ordering",
          "If Blob storage → ask about file patterns and formats",
      ]
  }

  Fabric Architect's Checklist

  FABRIC_ARCHITECT_REQUIREMENTS = {
      "must_know": [
          "Source type (from Source Analyst)",
          "Transformation needs (from Transform Expert)",
          "Destination type (tables/files)",
          "Refresh frequency",
      ],
      "should_know": [
          "Data volume for sizing",
          "Parallelism requirements",
          "Error handling preferences",
      ],

      "decision_matrix": {
          "Copy Activity": {
              "when": ["No transforms", "Simple copy", "< 10 tables"],
              "avoid_when": ["Complex logic", "Need PII masking"],
          },
          "Dataflow Gen2": {
              "when": ["Light transforms", "Visual debugging helpful", "PII masking"],
              "avoid_when": ["Very large data", "Complex Python logic"],
          },
          "Notebook": {
              "when": ["Complex transforms", "ML logic", "Custom Python", "> 100GB"],
              "avoid_when": ["Simple copies", "Non-technical team"],
          },
          "Mirroring": {
              "when": ["Real-time CDC", "SQL Server/Azure SQL source"],
              "avoid_when": ["One-time migration", "Non-SQL sources"],
          },
      },

      "thinking_rules": [
          "If 20+ tables → use ForEach with parallelism",
          "If needs aggregations → recommend Gold layer",
          "If Power BI destination → optimize for DirectQuery or Import",
          "If real-time → consider Eventstream",
      ]
  }

  Key Design Principle: Agents Know What They Don't Know

  class AgentThinking:
      """Each agent follows this thinking pattern"""

      def process(self, context):
          # 1. What do I know?
          known = self.extract_known_info(context)

          # 2. What do I NEED to know to do my job?
          required = self.get_requirements_checklist()

          # 3. What's missing?
          missing = required - known

          # 4. Can I infer anything?
          inferred = self.apply_thinking_rules(known)

          # 5. What must I ask?
          must_ask = missing - inferred

          if must_ask:
              return self.generate_smart_questions(must_ask)
          else:
              return self.generate_recommendation(known + inferred)

  ---
  Summary: The "Thinking" Behavior

  | Behavior                      | How It Works                            |
  |-------------------------------|-----------------------------------------|
  | Understand context            | Discovery Agent asks WHY, not just WHAT |
  | Infer intelligently           | "Customer data" → likely has PII        |
  | Ask smart questions           | Combine multiple questions logically    |
  | Know when done                | Track requirements checklist            |
  | Provide expertise             | Each agent brings deep knowledge        |
  | Recommend, don't just collect | "I recommend X because Y"               |
